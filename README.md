# Telegram to Discord notifier

Desktop-приложение для Windows, которое работает как Telegram userbot, живет в системном трее и отправляет уведомления в Discord через Webhook при получении нового личного сообщения от Telegram-контакта.

## Возможности

- desktop UI на `PyQt5`
- сворачивание в системный трей Windows
- авторизация Telegram по QR-коду
- авторизация Telegram по номеру телефона, коду и 2FA-паролю
- запуск и остановка мониторинга из интерфейса
- уведомления в Discord через `Webhook URL`
- логирование в окно приложения и в `%AppData%\TgDsNotifier\logs\app.log`
- сохранение настроек в `%AppData%\TgDsNotifier\app_config.json`
- хранение Telegram-сессии в `%AppData%\TgDsNotifier\session\`

## Установка

```bash
pip install -r requirements.txt
```

## Где взять данные

### Telegram API

1. Перейдите на `https://my.telegram.org`.
2. Откройте `API development tools`.
3. Создайте приложение.
4. Скопируйте `api_id` и `api_hash`.

### Discord Webhook URL

1. Откройте Discord и создайте сервер или выберите существующий.
2. Откройте настройки нужного текстового канала.
3. Перейдите в `Интеграции` -> `Вебхуки`.
4. Создайте новый вебхук.
5. Нажмите `Copy Webhook URL` и вставьте адрес в приложение.

## Запуск desktop-приложения

```bash
python desktop_app.py
```

## Как пользоваться

1. Откройте вкладку `Настройки`.
2. Заполните `TG API ID`, `TG API Hash`, `Discord Webhook URL`.
3. При необходимости задайте `Session Name` и SOCKS5-прокси.
4. Нажмите `Сохранить настройки`.
5. Перейдите во вкладку `Авторизация`.
6. Выполните вход одним из способов.

### Вход по QR

1. Нажмите `Показать QR`.
2. В Telegram откройте `Настройки` -> `Устройства` -> `Подключить устройство`.
3. Отсканируйте QR-код.
4. Если включена двухэтапная защита, введите пароль 2FA в приложении.

### Вход по номеру телефона

1. Введите номер телефона в международном формате.
2. Нажмите `Получить код`.
3. Введите код из Telegram.
4. Если включена двухэтапная защита, введите пароль 2FA.

### Запуск мониторинга

1. После успешной авторизации нажмите `Старт`.
2. Приложение начнет слушать новые входящие личные сообщения.
3. Уведомления отправляются только для пользователей, которые есть в Telegram-контактах или являются взаимными контактами.
4. При закрытии окна приложение не завершится, а свернется в трей.
5. Для полного выхода используйте меню иконки в трее.

## Консольный режим

Для запуска без GUI доступен скрипт `tg_to_discord_userbot.py`.

```bash
python tg_to_discord_userbot.py
```

Он использует `.env` и поддерживает авторизацию по QR или по номеру телефона.

## Сборка в `.exe`

```bash
build_app.bat
```

Результат появится в папке `dist`.

Если нужно собрать вручную:

```bash
pyinstaller --noconsole --name TgDsNotifier desktop_app.py
```

## Хранение файлов

- настройки приложения: `%AppData%\TgDsNotifier\app_config.json`
- лог-файл: `%AppData%\TgDsNotifier\logs\app.log`
- сессия Telegram: `%AppData%\TgDsNotifier\session\tg_userbot_session.session`

## Миграция старых данных

- если рядом с приложением или в папке проекта уже есть старый `app_config.json`, приложение сможет прочитать его при первом запуске
- если в `%AppData%\TgVkNotifier` осталась старая Telegram-сессия, приложение использует ее как источник для мягкой миграции в новую папку `%AppData%\TgDsNotifier`
- старые файлы автоматически не удаляются

## Интерфейс

Во вкладке `Настройки` теперь используются только поля Telegram, `Discord Webhook URL`, имя сессии и прокси. Поля и опции, связанные с VK, удалены.

## Примечания

- `app_config.json` и `.env` не должны попадать в публичный репозиторий.
- Уведомления Discord работают только в одну сторону: приложение отправляет события, но не удаляет уже доставленные сообщения.
- Основной сценарий использования - desktop-режим, но консольный режим также сохранен.
